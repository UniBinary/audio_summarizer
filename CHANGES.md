# Audio Summarizer 1.1版本更新说明
- 上个版本：**Version 1 Alpha 1** - 1.0a1  

## 1.功能增强

### Checkpoint功能实现
- 上个版本：没有Checkpoint功能，无法在上次断点处继续处理。
- 现版本：实现Checkpoint功能，支持在断点处继续处理，提升工具的可用性和稳定性。

## 2. 支持

### 操作系统支持调整
- 上个版本：仅支持`Windows`系统
- 现版本：调整为跨平台支持，用户可以在`Windows`、`macOS`和`Linux`系统上使用该工具。

## 3. 接口&调用

### 接口调整
- 上个版本：接口混乱，当需要参数传入路径时，接受的类型可能为`str`，也可能为`pathlib.Path`，在类/函数内也没有统一的类型，导致代码可读性和维护性较差。
- 现版本：接口调整为统一接受`Union[str, pathlib.Path]`类型的路径参数，并在类/函数内统一使用`pathlib.Path`，仅在需要时转换为`str`，提升代码的可读性和维护性。

### 主函数调用方式处理
- 上个版本：主函数的两种调用方式（直接调用和命令行调用）的参数处理不当，导致代码混乱。
- 现版本：调整主函数对不同调用方式的参数处理逻辑，使用额外的函数处理命令行调用的参数，提升代码的可读性和维护性。

### 主函数参数调整
- 上个版本：主函数参数过多，可读性和维护性较差。
- 现版本：调整主函数参数，减少参数数量，并将一些不常变动的参数封装到配置中，提升代码的可读性和维护性。

## 4. 日志&调试

### 日志记录调整
- 上个版本：日志的来源不够明确，不知道是哪个类或函数在记录日志。
- 现版本：调整日志记录的格式，在日志消息的开头添加每个类自己的标签，明确日志来源，提升日志的可读性和维护性。

### 5. Bug修复
- 修复了在音频转文字步骤中可能出现的编号错乱问题，确保生成的文本文件中的编号与原始音频文件中的编号一致。
- 修复了在音频提取步骤中可能出现的警告误报问题，确保工具在提取音频文件时不会产生不必要的警告信息。
